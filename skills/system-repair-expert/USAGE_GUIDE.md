# FixClawd 使用指南

## 如何使用

当您遇到系统、软件、配置或错误问题时，可以调用此技能。技能会严格按照六步法进行问题诊断和修复：

### 1. 提供尽可能详细的问题信息
- 完整的错误信息
- 相关日志片段
- 操作系统和软件版本
- 问题发生的上下文和重现步骤

### 2. 技能将自动执行以下步骤：

#### ✅ 第1步：问题理解
技能会首先分析您提供的信息并总结问题核心。

#### → 第2步：官方方案查询
搜索相关软件的官方文档和解决方案。

#### → 第3步：ClawdHub技能匹配
检查是否有现成的技能可以直接解决问题。

#### → 第4步：创建新技能判断
如果问题具有通用性，可能会建议创建新的可复用技能。

#### → 第5步：GitHub方案搜索
查找社区中的解决方案。

#### → 第6步：编写修复脚本（最后手段）
仅在前面方法都无效时，才会考虑编写一次性修复脚本。

## 示例对话

**用户**: "nginx启动失败，显示502错误"

**助手**: 
"✅ 第1步：问题理解完成 → Nginx服务启动失败并返回502错误码

→ 第2步：官方方案查询中...正在查找Nginx官方文档中关于502错误的解决方案...

需要确认：请提供更多详细信息：
1. 完整的错误日志
2. nginx配置文件的相关部分
3. 您的操作系统版本
4. 最近对系统或nginx配置的更改"

## 注意事项

- 请提供尽可能详细的错误信息，以便更准确地诊断问题
- 技能会优先寻找官方和经过验证的解决方案
- 对于复杂问题，可能需要多轮交互来收集必要的信息
